5. Project Idea

Overview

The project idea I have chosen is regarding the application of IT, namely: home automation. Soon enough, everything will be automated and establishing a reliable and universal set standard will allow multiple industry protocols to talk to a common interface. The idea for this project is to create a single device that can communicate to well-known protocols allowing for cross-protocol communication and creating hybrid environments where multiple services and protocols exist simultaneously. Most automated machines today rely on their own protocol, and getting different branded devices to communicate with each other can be challenging.

Motivation

Everyone would love to simplify their tedious day-to-day chores and responsibilities, and often, these things can become burdensome and stressful. A majority of tasks can quickly be done through a form of automation. Some examples include ordering groceries based on what is missing from your pantry/fridge, turning off lights when they are not in use, and reminding homeowners that they need to take the rubbish out on bin day. There are multiple benefits of such simple automations, including energy saving, physical security, and maximising efficient use of resources, to name a few.

Description

Multiple devices, protocols and services to automate homes are available on the market. Each of these provides a walled ecosystem forcing the homeowner to use one product over another, even when a better solution exists in a competing ecosystem. As the automated home increases in popularity, these walled ecosystems become limiting in functionality, ease of use, budget or all of the above. The ability to use multiple ecosystems simultaneously will greatly improve the adoption of home automation and simplify its use for the common household. This would allow homeowners to simply purchase any of the standardised home automation devices based on standard protocols and be sure that they will interoperate with this project's resultant product. Perhaps the most crucial feature of this system will be ability to interface with multiple ecosystems from a single pane of glass and a single application rather than relying on multiple applications for each walled garden. The RPi will be connected to home ethernet networks and can also be accessed via a simple web browser on both desktop and mobile devices. The aim of this centralised hub is to simplify home automation through easy-to-use guided steps that walk through the appropriate steps to create an automation. This could later be developed to use AI to optimise automation further and increase energy savings by learning the homeowners' habits and pre-empting their needs. An example of this would be to turn on the air conditioning before the homeowner arrives home and when the house temperature exceeds a set limit based on the homeowners' previous inputs. As previously stated, the tools and technologies of this project will mainly consist of an RPi, and how the user interacts with the interface will be up to them. The RPi has been chosen due to its compact design and relatively cheap pricing. The dashboard will be accessible via a simple web page. As long as the user is connected to the internet, this web page can be viewed from anywhere on any device that has a web browser.

Tools and Technologies

One of the main elements of this project will be a microcomputer called Rasberry Pi (RPi). This RPi will be able to connect to various branded automation-equipped products and will translate them into a single service that can be used to create cross-platform automation. The RPi will serve as a hub accessible via a simple webpage that can be accessed on a mobile phone or desktop device. This hub can be personalised to the user's preferred layout and will be based on freely available open-source software.

Skills Required

For the hub to perform its duties, it will need to be able to talk to the various protocols present in the automation devices. Many of these protocols are open standards with known programming interfaces. This project will need to take advantage of these programming interfaces and will require knowledge of each protocol or standard being used. Since these protocols or standards are open, it should not be difficult to find the necessary skills needed to develop such a solution. The aim of using a small device such as a Raspberry Pi also makes it accessible to most people.

Outcome

If this project succeeds, homeowners will be able to integrate a number of industry-standard protocols into a single centralised hub, allowing each protocol or service to communicate seamlessly with other protocols. Homeowners will have a much better grasp of how mundane, day-to-day tasks affect their lives. In addition, homeowners will be able to keep a close eye on their environmental impact through energy monitoring and reduce their impact through simple electronic devices, which automate the process of, for example, turning devices off when they are not in use. Homeowners will no longer be tied to a single ecosystem allowing them to create complex cross-platform automation whereby multiple protocols and services can co-exist.
